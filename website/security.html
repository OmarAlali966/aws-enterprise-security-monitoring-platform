How We Secure This Site

We practice what we sell. This page documents the real controls protecting this environment.

Identity & Access

All infrastructure changes are made through named IAM roles scoped to least privilege. The AWS account root user is not used for daily operations, is protected with multi-factor authentication, and has no active access keys.

Logging & Audit Trail

AWS CloudTrail records every API call made in this account, including who made it, from where, and what it changed. The trail is enabled across every AWS region and captures global service events, so activity cannot hide in a region the team does not normally monitor. Every delivered log file is cryptographically validated so any tampering can be detected, encrypted at rest with a dedicated customer-managed KMS key, and stored in a versioned S3 bucket that blocks all public access and rejects any request that is not sent over HTTPS.

Threat Detection

Amazon GuardDuty continuously analyzes account activity, network traffic metadata, and DNS queries for indicators of compromise such as compromised credentials, cryptomining, and reconnaissance against our AWS resources. Findings are automatically aggregated in AWS Security Hub, which also runs the AWS Foundational Security Best Practices standard against every resource in the account and assigns an overall compliance score.

Configuration Compliance

AWS Config continuously records the configuration of every supported resource in this account and evaluates it against a set of automated rules, including checks for public S3 buckets, unencrypted storage, overly permissive IAM policies, and security groups left open to the internet. Any resource that drifts out of compliance is flagged immediately rather than being discovered during a periodic manual review.

Data Protection

Data at rest is encrypted using AWS Key Management Service (KMS) with customer-managed keys, each governed by a key policy that names exactly which accounts, services, and roles may use it. The website content bucket blocks public write access, and public read access is limited to the specific objects required to serve the site.

Alerting & Response

High-severity findings trigger automated notifications through Amazon SNS so they can be triaged quickly rather than discovered during a periodic review.

Responsible Disclosure

This is a portfolio and demonstration environment rather than a production system handling real customer data. If you believe you have found a genuine issue with how this site is configured, please reach out through the contact page.
