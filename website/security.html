How We Secure This Site

We practice what we sell. This page documents the real controls protecting this environment.

Identity & Access

All infrastructure changes are made through named IAM roles scoped to least privilege. The AWS account root user is not used for daily operations, is protected with multi-factor authentication, and has no active access keys.

Logging & Audit Trail

AWS CloudTrail records every API call made in this account, including who made it, from where, and what it changed. Logs are delivered to a dedicated, encrypted, access-restricted S3 bucket and retained for audit purposes.

Threat Detection

Amazon GuardDuty continuously analyzes account activity, network traffic metadata, and DNS queries for indicators of compromise. Findings are aggregated in AWS Security Hub alongside configuration compliance checks from AWS Config.

Data Protection

Data at rest is encrypted using AWS Key Management Service (KMS) with customer-managed keys. The website content bucket blocks public write access, and public read access is limited to the specific objects required to serve the site.

Alerting & Response

High-severity findings trigger automated notifications through Amazon SNS so they can be triaged quickly rather than discovered during a periodic review.

Responsible Disclosure

This is a portfolio and demonstration environment rather than a production system handling real customer data. If you believe you have found a genuine issue with how this site is configured, please reach out through the contact page.
